# 基本設計書

## 1. システム概要

### 1-1. 目的

Windows環境における不要ファイルやレジストリエントリーを検出・削除し、  
ディスク容量の回復とシステムの整理を行うCLIツール。

### 1-2. 基本方針

- 軽量かつポータブル：PowerShell 5.1のみで動作し、追加インストール不要
- 安全性重視：削除前に分析結果を表示し、ユーザー確認を必須とする
- 拡張性：SOLID原則に基づくモジュール設計で、新機能の追加が容易

### 1-3. 対象ユーザー

PowerShellを実行可能なWindowsユーザー。

## 2. 動作環境

| 項目       | 要件                                                    |
| ---------- | ------------------------------------------------------- |
| OS         | Windows 10 / 11                                         |
| PowerShell | 5.1以上（Windows標準搭載）                              |
| 権限       | 一時ファイル削除：一般ユーザー / レジストリ操作：管理者 |

## 3. システム構成

```text
win-cleaner/
├── win-cleaner.ps1              # エントリーポイント
├── modules/
│   ├── Core/                    # 共通基盤
│   │   ├── ICleanerModule.psm1
│   │   ├── CleanerEngine.psm1
│   │   └── PermissionChecker.psm1
│   ├── TempCleaner/             # 一時ファイル削除
│   │   ├── TempCleaner.psm1
│   │   └── TempCleanerRule.psm1
│   └── RegistryCleaner/         # レジストリクリーナー
│       ├── RegistryCleaner.psm1
│       └── RegistryCleanerRule.psm1
├── config/
│   └── settings.json            # 設定ファイル
└── tests/                       # Pesterテスト
```

## 4. モジュール構成

### 4-1. Core（共通基盤）

全クリーナーモジュールが依存する共通層。

| モジュール        | 責務                                           |
| ----------------- | ---------------------------------------------- |
| ICleanerModule    | クリーナーの抽象基底クラス・共通データ型の定義 |
| CleanerEngine     | モジュールの登録・実行制御・結果集約           |
| PermissionChecker | 管理者権限の検証                               |

### 4-2. TempCleaner（一時ファイル削除）

| モジュール      | 責務                                           |
| --------------- | ---------------------------------------------- |
| TempCleaner     | ICleanerModuleを継承した一時ファイル削除の実装 |
| TempCleanerRule | 削除対象パスの展開・除外パターン判定           |

### 4-3. RegistryCleaner（レジストリクリーナー）

| モジュール          | 責務                                               |
| ------------------- | -------------------------------------------------- |
| RegistryCleaner     | ICleanerModuleを継承したレジストリクリーナーの実装 |
| RegistryCleanerRule | 不要レジストリキーの判定ルール                     |

## 5. 処理フロー

### 5-1. 全体フロー

```text
起動
  ├── 設定ファイル読み込み（settings.json）
  ├── モジュール登録（依存注入）
  └── メインループ
        ├── メニュー表示
        ├── ユーザー選択
        ├── 権限チェック（必要な場合）
        ├── 分析（Analyze）
        │     └── 検出結果の表示
        ├── ユーザー確認（y/N）
        ├── クリーニング実行（Clean）
        │     └── 結果表示（削除数・解放容量・エラー）
        └── メニューに戻る or 終了
```

### 5-2. 分析・実行の2段階方式

安全性を確保するため、全モジュールで以下の2段階を踏む。

1. **Analyze**: 削除対象を検出し一覧として返す（ファイルシステム・レジストリは変更しない）
2. **Clean**: Analyzeの結果を受け取り、実際の削除を実行する

## 6. SOLID原則の適用

| 原則                        | 適用方法                                                 |
| --------------------------- | -------------------------------------------------------- |
| 単一責任（SRP）             | 各モジュールが1つの責務のみ担当                          |
| 開放閉鎖（OCP）             | 新しいクリーナーはICleanerModuleを継承するだけで追加可能 |
| リスコフ置換（LSP）         | 全クリーナーが共通インターフェイスに従い差し替え可能     |
| インターフェイス分離（ISP） | ICleanerModuleは最小限のメソッドのみ定義                 |
| 依存性逆転（DIP）           | CleanerEngineは具象クラスではなくICleanerModuleに依存    |

## 7. セキュリティ設計

### 7-1. 権限制御

レジストリ操作は管理者権限を必須とし、起動時に検証する。

### 7-2. 削除の安全性

- 削除前に必ずユーザー確認を行う
- システムファイル（`.sys`、`.dll`）はデフォルトで除外する
- 削除に失敗したファイルはエラーとして記録し、処理を継続する
