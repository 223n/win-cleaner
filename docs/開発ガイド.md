# 開発ガイド

Windows Cleanerの開発に参加するための情報をまとめたドキュメントです。

## 開発環境のセットアップ

### 必須ツール

- PowerShell 5.1以上（Windows標準搭載）またはPowerShell 7+
- Git

### 推奨ツール

```powershell
# Pester（テストフレームワーク）
Install-Module -Name Pester -Force -SkipPublisherCheck

# PSScriptAnalyzer（静的解析）
Install-Module -Name PSScriptAnalyzer -Force
```

### リポジトリのクローン

```powershell
git clone https://github.com/223n/win-cleaner.git
cd win-cleaner
```

### 動作確認

```powershell
# テスト実行
Invoke-Pester -Path .\tests\ -Output Detailed

# 静的解析
Invoke-ScriptAnalyzer -Path . -Recurse -Settings .\PSScriptAnalyzerSettings.psd1
```

## アーキテクチャ概要

### モジュール構成

```text
modules/
├── Core/                       # 共通基盤
│   ├── ICleanerModule.psm1     # モジュールインターフェース
│   ├── CleanerEngine.psm1      # モジュール管理エンジン
│   ├── SettingsValidator.psm1  # 設定ファイル検証
│   ├── PermissionChecker.psm1  # 管理者権限チェック
│   └── Logger.psm1             # ログ出力
├── TempCleaner/                # 一時ファイル削除
│   ├── TempCleaner.psm1
│   └── TempCleanerRule.psm1
└── RegistryCleaner/            # レジストリクリーナー
    ├── RegistryCleaner.psm1
    └── RegistryCleanerRule.psm1
```

### ICleanerModuleインターフェイス

すべてのクリーナーモジュールは`ICleanerModule`クラスを継承します。

```powershell
class ICleanerModule {
    [string] GetName()                          # モジュール名を返す
    [string] GetDescription()                   # 説明を返す
    [bool] RequiresAdmin()                      # 管理者権限が必要か
    [CleanerItem[]] Analyze()                   # 対象を検出する
    [CleanerResult] Clean([CleanerItem[]]$items) # 検出結果を削除する
}
```

### データクラス

```powershell
# 検出された個別のアイテム
class CleanerItem {
    [string]$Path          # ファイルパスまたはレジストリパス
    [long]$Size            # サイズ（バイト）
    [string]$Category      # 分類名（表示用）
    [string]$PropertyName  # レジストリ値の名前（省略可）
}

# クリーニング結果
class CleanerResult {
    [int]$ItemCount        # 削除成功数
    [long]$FreedBytes      # 解放したバイト数
    [string[]]$Errors      # エラーメッセージ一覧
}
```

### 処理フロー

1. `win-cleaner.ps1`がモジュールを`CleanerEngine`に登録する
1. ユーザーがメニューからモジュールを選択する
1. `Analyze()`で対象を検出し、結果を表示する
1. ユーザーが`y/N`で確認する
1. `Clean()`で検出結果を削除する

Analyze（検出）とClean（削除）を分離することで、
削除前の確認を確実に行えます。

### 設定ファイルの構造

`config/settings.json`はモジュール名をキーとするJSONです。

```json
{
  "tempCleaner": {
    "targets": [...],
    "excludePatterns": [...]
  },
  "registryCleaner": {
    "targets": [...]
  }
}
```

設定ファイルは起動時に`SettingsValidator`で検証されます。
必須フィールドの欠落や無効なルール名はエラーとなります。

## 新しいモジュールの追加方法

新しいクリーナーモジュールを追加する手順です。
ここでは`DiskCleaner`という架空のモジュールを例にします。

### 手順1: モジュールの実装

`modules/DiskCleaner/DiskCleaner.psm1`を作成します。

```powershell
using module ..\Core\ICleanerModule.psm1

class DiskCleaner : ICleanerModule {
    [hashtable]$Settings

    DiskCleaner([hashtable]$settings) {
        $this.Settings = $settings
    }

    [string] GetName() {
        return "Disk Cleaner"
    }

    [string] GetDescription() {
        return "Detect and remove unnecessary disk files"
    }

    [bool] RequiresAdmin() {
        return $false
    }

    [CleanerItem[]] Analyze() {
        $items = [System.Collections.Generic.List[CleanerItem]]::new()
        # 検出ロジックを実装
        return $items.ToArray()
    }

    [CleanerResult] Clean([CleanerItem[]]$items) {
        $result = [CleanerResult]::new()
        foreach ($item in $items) {
            try {
                Remove-Item -Path $item.Path -Force -ErrorAction Stop
                $result.ItemCount++
                $result.FreedBytes += $item.Size
            }
            catch {
                $result.Errors += "Failed to delete: $($item.Path)"
            }
        }
        return $result
    }
}
```

### 手順2: 設定ファイルへの追加

`config/settings.json`にモジュール用のセクションを追加します。

```json
{
  "diskCleaner": {
    "targets": [
      {
        "category": "Old Downloads",
        "path": "%USERPROFILE%\\Downloads",
        "pattern": "*.tmp",
        "recurse": false
      }
    ]
  }
}
```

### 手順3: SettingsValidatorへの検証ルール追加

`modules/Core/SettingsValidator.psm1`の`Test-SettingsSchema`関数に
新しいセクションの検証ロジックを追加します。

### 手順4: エントリーポイントへの登録

`win-cleaner.ps1`に`using module`と`Register`呼び出しを追加します。

```powershell
# ファイル先頭に追加
using module .\modules\DiskCleaner\DiskCleaner.psm1

# Start-WinCleaner関数内に追加
$engine.Register([DiskCleaner]::new($settings))
```

### 手順5: テストの作成

`tests/DiskCleaner.Tests.ps1`を作成し、
最低限以下のテストを実装します。

- モジュールのプロパティ（名前、管理者権限）
- Analyzeが空の結果を正しく返すこと
- Cleanがエラーを正しく処理すること

## テスト

### テストの実行

```powershell
# 全テスト実行
Invoke-Pester -Path .\tests\ -Output Detailed

# 特定のテストファイルのみ実行
Invoke-Pester -Path .\tests\RegistryCleaner.Tests.ps1 -Output Detailed
```

### テストの構造

テストはPesterフレームワークの
`Describe`/`Context`/`It`パターンで記述します。

```powershell
#Requires -Modules Pester
using module ..\modules\Core\ICleanerModule.psm1
using module ..\modules\YourModule\YourModule.psm1

BeforeAll {
    Import-Module "$PSScriptRoot\..\modules\YourModule\YourModule.psm1" -Force
}

Describe "YourModule" {
    Context "Module properties" {
        It "should return correct name" {
            # テスト内容
        }
    }
}
```

### テスト時の注意事項

- `using module`はファイル先頭に記述する（PowerShellの制約）
- `Import-Module -Force`で最新のコードを確実に読み込む
- レジストリテストではテスト用キーを`try/finally`で必ず削除する
- テスト用キーは`HKCU:\SOFTWARE\`配下にランダム名で作成し、
  他のテストとの競合を防ぐ

## コーディング規約

### 命名規則

- 関数: `Verb-Noun`形式（PowerShell標準の動詞を使用）
  - 例: `Test-InvalidRegistryReference`、`Get-ExecutablePath`
- クラス: PascalCase
  - 例: `TempCleaner`、`CleanerEngine`
- 変数: camelCase
  - 例: `$items`、`$baseKey`

### エラーハンドリング

権限不足が想定されるレジストリ操作では、
`SecurityException`と`UnauthorizedAccessException`を
明示的にcatchします。

```powershell
try {
    # レジストリ操作
}
catch [System.Security.SecurityException],
      [System.UnauthorizedAccessException] {
    # 権限不足 — スキップ
}
catch {
    Write-Warning "エラー: $($_.Exception.Message)"
}
```

### レジストリ操作

- パフォーマンスが重要な場面では、
  PowerShellプロバイダー（`Test-Path`、`Get-ChildItem`）ではなく
  .NET API（`OpenSubKey`、`GetSubKeyNames`）を使用する
- `OpenSubKey`で取得したキーは必ず`finally`ブロックで
  `Dispose()`する

```powershell
$key = $null
try {
    $key = $hive.OpenSubKey($subPath, $false)
    if ($null -ne $key) {
        # キーを使った操作
    }
}
finally {
    if ($null -ne $key) { $key.Dispose() }
}
```

### Export-ModuleMember

- モジュールの末尾で公開する関数を明示的に指定します。
- クラスのみを公開するモジュールでも空配列で記述します。

```powershell
# 関数を公開する場合
Export-ModuleMember -Function @('Test-InvalidRegistryReference',
                                'Get-ExecutablePath')

# クラスのみの場合
Export-ModuleMember -Function @()
```
