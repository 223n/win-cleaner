# 詳細設計書

## 1. クラス設計

### 1-1. 共通データ型

#### 1-1-1. CleanerItem

検出された削除対象1件を表すデータクラス。

| プロパティ | 型 | 説明 |
| --- | --- | --- |
| Path | string | 対象のフルパス（ファイルパスまたはレジストリパス） |
| Size | long | サイズ（バイト）。レジストリの場合は0 |
| Category | string | 分類名（表示用） |

#### 1-1-2. CleanerResult

クリーニング実行結果を表すデータクラス。

| プロパティ | 型 | 説明 |
| --- | --- | --- |
| ItemCount | int | 削除成功件数 |
| FreedBytes | long | 解放した合計バイト数 |
| Errors | string[] | 削除失敗時のエラーメッセージ一覧 |

### 1-2. 抽象基底クラス（ICleanerModule）

全クリーナーモジュールが継承する基底クラス。

| メソッド | 戻り値 | 説明 |
| --- | --- | --- |
| GetName() | string | モジュール名を返す |
| GetDescription() | string | モジュールの説明を返す |
| RequiresAdmin() | bool | 管理者権限が必要かどうかを返す |
| Analyze() | CleanerItem[] | 削除対象を検出して返す（副作用なし） |
| Clean(CleanerItem[]) | CleanerResult | 指定されたアイテムを削除し結果を返す |

### 1-3. CleanerEngine

モジュールの登録と実行を制御するエンジンクラス。

| メソッド | 戻り値 | 説明 |
| --- | --- | --- |
| Register(ICleanerModule) | void | クリーナーモジュールを登録する |
| GetModuleList() | hashtable[] | 登録済みモジュールの一覧を返す |
| AnalyzeModule(int) | CleanerItem[] | 指定インデックスのモジュールで分析を実行する |
| CleanModule(int, CleanerItem[]) | CleanerResult | 指定インデックスのモジュールでクリーニングを実行する |

### 1-4. Logger

実行ログをファイルに記録するクラス。

| プロパティ | 型 | 説明 |
| --- | --- | --- |
| LogDir | string | ログ出力ディレクトリのパス |
| LogPath | string | 現在のセッションのログファイルパス |

| メソッド | 引数 | 説明 |
| --- | --- | --- |
| Write | message(string) | タイムスタンプ付きでメッセージを記録する |
| WriteAnalyzeResult | moduleName(string), items(object[]) | 分析結果をカテゴリごとに記録する |
| WriteCleanResult | moduleName(string), result(object) | クリーニング結果とエラーを記録する |
| WriteDryRun | moduleName(string), items(object[]) | ドライラン時の削除対象一覧を記録する |

**ログファイル形式:**

- ファイル名：`win-cleaner_YYYYMMDD_HHmmss.log`
- 出力先：`logs/`ディレクトリ（自動作成）
- 各行の形式：`[YYYY-MM-DD HH:mm:ss] メッセージ`

## 2. モジュール詳細

### 2-1. TempCleaner

#### 2-1-1. TempCleaner.psm1

ICleanerModuleを継承。設定ファイルで定義された対象パスに対し、ファイルの検出と削除を行う。

**Analyze処理:**

1. `Get-TempCleanerTargets`で対象パス一覧を取得する
2. 各パスに対して`Get-ChildItem`でファイルを列挙する
3. `Test-ExcludedPath`で除外パターンに該当するファイルを除外する
4. 該当ファイルを`CleanerItem`として返す

**Clean処理:**

1. 各`CleanerItem`に対して`Remove-Item -Force`を実行する
2. 成功時は`ItemCount`と`FreedBytes`を加算する
3. 失敗時はエラーメッセージを`Errors`に追加し、処理を継続する

#### 2-1-2. TempCleanerRule.psm1

| 関数 | 引数 | 戻り値 | 説明 |
| --- | --- | --- | --- |
| Get-TempCleanerTargets | Settings(hashtable) | hashtable[] | 設定から対象パスを展開して返す |
| Test-ExcludedPath | FilePath(string), ExcludePatterns(string[]) | bool | ファイルが除外パターンに該当するか判定する |

#### 2-1-3. デフォルト削除対象

| カテゴリ | パス | 再帰 |
| --- | --- | --- |
| Windows Temp | `%TEMP%` | あり |
| Windows Temp | `%SystemRoot%\Temp` | あり |
| Windows Prefetch | `%SystemRoot%\Prefetch` | なし |
| Windows Update Cache | `%SystemRoot%\SoftwareDistribution\Download` | あり |
| Chrome Cache | `%LOCALAPPDATA%\Google\Chrome\User Data\Default\Cache\Cache_Data` | あり |
| Chrome Cache | `%LOCALAPPDATA%\Google\Chrome\User Data\Default\Code Cache` | あり |
| Edge Cache | `%LOCALAPPDATA%\Microsoft\Edge\User Data\Default\Cache\Cache_Data` | あり |
| Edge Cache | `%LOCALAPPDATA%\Microsoft\Edge\User Data\Default\Code Cache` | あり |
| Firefox Cache | `%LOCALAPPDATA%\Mozilla\Firefox\Profiles\*\cache2\entries` | なし |
| Brave Cache | `%LOCALAPPDATA%\BraveSoftware\Brave-Browser\User Data\Default\Cache\Cache_Data` | あり |
| Brave Cache | `%LOCALAPPDATA%\BraveSoftware\Brave-Browser\User Data\Default\Code Cache` | あり |

### 2-2. RegistryCleaner

#### 2-2-1. RegistryCleaner.psm1

ICleanerModuleを継承。設定ファイルで定義されたレジストリキーを走査し、無効なエントリーの検出と削除を行う。

**Analyze処理:**

1. `Get-RegistryCleanerTargets`で対象キー一覧を取得する
2. 各キーのサブキーを`Get-ChildItem`で列挙する
3. ルールに応じた判定を行う
   - `invalidFileReference`: デフォルト値が存在しないファイルパスを参照している
   - `invalidAppPath`: Pathプロパティが存在しないファイルパスを参照している
4. 該当エントリーを`CleanerItem`として返す

**Clean処理:**

1. 各`CleanerItem`に対して`Remove-Item -Recurse -Force`を実行する
2. 成功時は`ItemCount`を加算する
3. 失敗時はエラーメッセージを`Errors`に追加し、処理を継続する

#### 2-2-2. RegistryCleanerRule.psm1

| 関数 | 引数 | 戻り値 | 説明 |
| --- | --- | --- | --- |
| Get-RegistryCleanerTargets | Settings(hashtable) | hashtable[] | 設定から対象レジストリキーを返す |
| Test-InvalidRegistryReference | Path(string) | bool | パスが無効なファイル参照かを判定する |

**Test-InvalidRegistryReferenceの判定ロジック:**

1. 空文字列の場合：無効（true）
2. ドライブレター（`X:\`）またはUNCパス（`\\`）で始まる場合：`Test-Path`で存在確認する
3. 上記以外（ファイルパスではない値）：有効と見なす（false）

### 5-2. ヘルパー関数

| 関数 | 説明 |
| --- | --- |

### 5-3. 依存注入

`Start-WinCleaner`内で設定を読み込み、各モジュールのコンストラクターに渡す。

```powershell
$settings = Read-Settings
$logger = [Logger]::new($scriptRoot)
$engine = [CleanerEngine]::new()
$engine.Register([TempCleaner]::new($settings))
$engine.Register([RegistryCleaner]::new($settings))
```

新しいモジュールを追加する場合は、この箇所に`Register`呼び出しを1行追加するだけで済む。

## 6. エラーハンドリング

### 6-1. 方針

- 個別ファイル・レジストリキーの削除失敗は`CleanerResult.Errors`に記録し、処理を継続する
- 設定ファイルの読み込み失敗やモジュールの初期化失敗は例外をスローして終了する
- アクセス拒否エラーは`SilentlyContinue`でスキップし、結果に含めない

### 6-2. エラー分類

| 分類 | 対応 | 例 |
| --- | --- | --- |
| 致命的エラー | 例外スローで終了 | settings.jsonが見つからない |
| 操作エラー | Errorsに記録して継続 | ファイル削除失敗（ロック中など） |
| アクセスエラー | サイレントスキップ | 権限不足でレジストリキーを読めない |
